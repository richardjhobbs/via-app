## Deviations from prior brand playbook

1. **Beverage is not apparel.** `supportsSizing: true` is misleadingly named ‚Äî it gates the per-variant import, not a literal clothing size chart. For Nolo, the "size" values are pack counts ("12 Cans"/"24 Cans"/"36 Cans"). This worked cleanly without code changes, but naming is overdue for a rename to `hasVariants` or similar.
2. **Edition size semantics.** Frey / UU / Clooudie / PassportADV are all limited-run apparel where `edition_size = stock at mirror time` is correct. For a restockable FMCG brand, this caps the drop too low. Landed on `edition_size = 100` per drop at user instruction. A proper restock flow (`brand-mirror.mjs --brand nolo --restock` or a dedicated script) will be needed before sales push against that cap.
3. **Enrichment without vision.** Nolo's Shopify `body_html` is unusually thin (one-line taglines). The real product data lives on dedicated content pages (`/pages/about-nolo2-0`, `/pages/faqs-1`, and the individual product pages). Precomputed JSON built by hand from those pages delivers richer agent-facing context than vision on a can-on-white-background image could have anyway. Zero LLM spend.
4. **Shipping deferral.** UK-only rule encoded as `internationalFlatUsd: null` + `shipsFromCountry: 'GB'`. Real rates (¬£4 standard, free over ¬£30, DHL ¬£6.10 next-day) surfaced in MCP `instructions` + concierge SOUL as human-readable notes rather than as structured flat rates, because the free-over-¬£30 threshold can't be expressed in the current flat-rate schema. Real-time Shopify quoting is the right answer and is roadmapped.
5. **Branch vs master.** Commit landed on `rename/drops-to-listings` which carries a pending `drops ‚Üí listings` rename across `db.ts` helpers. Master still uses the old names. Nolo works live because the backing tables + generic routes don't depend on the rename. Master cherry-pick deferred until the rename branch merges.
